# 微信小程序开发学习





# 一、背景

微信小程序开发可以分为两种方式，一种是**原生开发**，一种是**借助框架开发**。

> 原生开发

小程序原生开发框架是微信官网首次推出的，目的是为了让开发者在微信中开发具有原生APP的体验尽可能简洁高效。原生开发不支持工程化，需要自己动手搭建;在语法规范上，使用的是[小程序开发](https://www.hooketech.com/)规范，没有组件化机制以及本身无自动构建。在开发方式上，需学习小程序的书写规范，开发成本较高，且原生开发不支持多端复用。

> 框架开发

形容 `wepy`、`mpvue`、`taro`、`uni-app` 等等框架都可以封装了原生行为，但里边还是会有些差异，具体体现在：

- 框架语法：wepy、mpvue、uni-app 都是基于 vuejs 开发，所以前置需要了解 vue 类似语法
- 框架跨端支持范围：wepy、mpvue 都仅支持跨 app 的小程序，而 taro、uni-app 支持 app 以及 h5

- 框架开发局限：这里每个框架或多或少都会有些使用上的局限性，比如在组件动态插槽以及数据处理中，这里先不进行赘述，等后续针对某项技术深入学习后再了解



# 二、基础知识

> 学习思路：
>
> 1. 粗略了解原生开发里边的知识点，
> 2. 选定一个 stars 数、社区认可度比较高的框架深入学习，可以保证在基础知识扎实的情况下进入框架学习



## 生命周期

![img](https://res.wx.qq.com/wxdoc/dist/assets/img/page-lifecycle.2e646c86.png)

上图是小程序页面生命周期，视图线程和应用处理线程通过 notify 与 send data 的方式互相通知执行相应回调

- onLoad：页面创建时执行
- onShow：页面出现在前台时执行
- onReady：页面渲染完成时执行
- onHide：页面从前台变为后台时执行
- onUnload：页面卸载时执行





## 原理考究



> Q：小程序与普通网页开发有什么开发

[面试官：说说微信小程序的实现原理？](https://github.com/febobo/web-interview/issues/289)：核心点在传统 web 开发里边 JS 引擎其实和 UI 渲染是互斥关系，主要体现在 [[浏览器中的JS引擎和渲染线程在主线程上互斥机制]](https://www.cnblogs.com/openmind-ink/articles/18648441)，而在小程序中渲染和逻辑是分开在两个线程中，两者是独立工作，因此比较少出现因为逻辑层高耗时任务阻塞渲染的任务。因为是分离，所以两者是通过线程通信的方式（微信客户端中转）交互任务，所以基本是走任务回调方式。![img](https://res.wx.qq.com/wxdoc/dist/assets/img/4-1.ad156d1c.png)
